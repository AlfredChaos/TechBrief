# TechBrief MVP 验收清单（P0）

更新时间：2026-05-22

## 1. 验收目标

- [ ] OpenAI 与 Anthropic 各选 5 篇历史文章回放处理，10/10 成功生成草稿（允许每篇 1 次手动重试）
- [ ] 标题层级、代码块、主要图片不丢失（抽样结构保真 ≥ 98%）
- [ ] 新文章发现：在模拟新增或实际新增情况下，系统能在调度周期内发现并入库
- [ ] 去重：同一文章重复巡检不会重复创建草稿
- [ ] 可追溯：任一失败能看到失败阶段与错误摘要；能从失败点重试并成功
- [ ] 文末署名模板在草稿中正确出现，包含来源/作者（如有）/原文链接/发布时间（如可得）/免责声明

## 2. P0 功能清单（逐项验收）

### 2.1 发现与去重

- [ ] RSS 优先；无 RSS 时轮询列表页
- [ ] 支持 ETag / Last-Modified 的条件请求
- [ ] canonical_url 优先；否则使用最终 URL
- [ ] 去重键：dedupe_key = sha256(source + canonical_url)
- [ ] 已处理文章不重复入队

### 2.2 抓取与落库

- [ ] 抓取详情页 HTML 成功，并保存 raw_html（含抓取时间、HTTP 状态、headers 摘要）
- [ ] raw_html 可用于追溯与重跑

### 2.3 正文抽取（结构化）

- [ ] 抽取结构单元：H1-H4、P、UL/OL、BLOCKQUOTE、CODE_BLOCK、IMAGE、HR
- [ ] 保留原文链接（URL 不变）
- [ ] 图片保留源 URL（后续由排版阶段处理上传与替换）
- [ ] 解析元信息：title、author（若可得）、published_at（若可得）、cover_image（若可得）

### 2.4 全文翻译（Minimax）

- [ ] code block / inline code 原样保留
- [ ] 链接 URL 不翻译；链接文本可翻译
- [ ] 段落与标题层级保持一致
- [ ] 支持 glossary（YAML/JSON）术语表并执行一致性替换
- [ ] 最小质量检查：标题数量与层级差异、code block 数量一致、关键链接数量一致

### 2.5 公众号排版与素材处理

- [ ] AST → 公众号富文本 HTML（尽量使用 p/h2/h3/strong/em/pre/code/img/blockquote/ul/ol/li）
- [ ] 图片处理：下载 → 去重（按内容 hash）→ 上传（或 mock）→ 替换 img src
- [ ] 默认策略：图片失败即阻断，并标记失败原因

### 2.6 草稿创建（接口层 + mock）

- [ ] 草稿创建适配层存在（真实接口可后续接入）
- [ ] 创建草稿后记录 wechat_draft_id（mock 模式也需要可追踪的 id）
- [ ] 若失败，记录错误码与响应摘要

### 2.7 状态、可观测、重试

- [ ] 每篇文章有处理状态、各阶段耗时、失败阶段与错误摘要
- [ ] 支持指数退避重试（至少覆盖翻译与公众号接口层）
- [ ] 支持从失败点重试（基于 checkpoint 或可恢复的中间产物）

### 2.8 文末固定追加

- [ ] 文末固定区块模板可配置
- [ ] 至少包含：来源站点名称、原文链接、免责声明；作者/发布时间可选（不可得时按规则省略或标记）
